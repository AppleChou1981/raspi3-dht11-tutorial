⚠️ 重要提醒：請先插妥杜邦線並確認所有接線正確後再行送電！若是電源（VCC）與接地（GND）接反，會導致 DHT11 及 Raspberry Pi 晶片過熱燒毀！
💡 架構導讀（方案 A）： 本教學採用「方案 A」。由樹莓派定時讀取 DHT11 後，轉化為 HTTP POST 請求並帶入指定的試算表 ID 發送給 Google Apps Script (GAS)。GAS 負責將數據寫入試算表，並由 **GAS 後台直接呼叫 LINE API** 發送通知，實現雲端中轉的整合架構。

如何檢索並提取 Google 試算表 ID？

打開你的 Google 雲端試算表，觀察瀏覽器上方網址列，結構如下圖。請僅複製兩條斜線中間的那一長串英數隨機亂碼：

https://docs.google.com/spreadsheets/d/【 只要複製這整串隨機亂碼，這就是試算表 ID 】/edit#gid=0

💡 註：這串 ID 稍後必須貼進 Python 程式碼中的 SPREADSHEET_ID 欄位。

2
硬體腳位與型號提醒（三針腳 vs 四針腳）

⚠️ 型號與電阻關鍵確認：

  • 3 腳位模組（常見藍色小板）： 通常已內建上拉電阻，因此通常不需外接電阻。接線時只需將模組的 VCC、GND、DATA 依序連接樹莓派即可。
  • 4 腳位元件（裸件）： 內部沒有整合電阻，必須在 VCC 與 DATA 之間手動外接一個 4.7kΩ ~ 10kΩ 的上拉電阻，否則樹莓派可能因訊號浮動而無法正確讀取到溫濕度數據（第 3 腳通常為空腳不接）。

Raspberry Pi 40-pin 實體排針接法對照表（樹莓派 3）：

左側功能標記 (奇數針腳) 實體 實體 右側功能標記 (偶數針腳)
接 DHT11 的 VCC（標示 + 或 VCC） ── [3.3V 電源 / Pin 1] 1 2 5V 電源
GPIO 2 (SDA) 3 4 5V 電源
GPIO 3 (SCL) 5 6 接 DHT11 的 GND（標示 - 或 GND） ── [Ground 地線 / Pin 6]
接 DHT11 的 DATA（標示 S、OUT 或 DAT） ── [GPIO 4 / Pin 7] 7 8 GPIO 14 (TXD)
Ground (接地) 9 10 GPIO 15 (TXD)

雲端後台：Google Apps Script 接收與 LINE 發送邏輯設定

請在 Google 試算表的「擴充功能」>「Apps Script」中貼上以下指令碼。此處已包含將數據寫入試算表，以及**同步發送動作至 LINE 官方 API** 的完整執行邏輯：

function doPost(e) {
  // 1. 解析來自樹莓派的 JSON 數據
  var params = JSON.parse(e.postData.contents);
  var targetId = params.spreadsheet_id; 
  var temp = params.temperature;
  var humi = params.humidity;
  
  // 2. 執行動作：開啟指定試算表並寫入歷史紀錄
  var ss = SpreadsheetApp.openById(targetId);
  var sheet = ss.getSheets()[0];
  var now = Utilities.formatDate(new Date(), "Asia/Taipei", "yyyy-MM-dd HH:mm:ss");
  sheet.appendRow([now, temp, humi]);
  
  // 3. LINE Bot 發送邏輯：設定憑證與傳輸內容
  var LINE_ACCESS_TOKEN = "請在此處貼上你的_LINE_Channel_Access_Token";
  var TARGET_USER_ID = "請在此處貼上你的_LINE_Your_user_ID";
  
  var line_api_url = "https://api.line.me/v2/bot/message/push";
  var message_text = "\n🌡️ 樹莓派雲端 GAS 回報\n-------------------\n⏰ 時間：" + now + "\n°C 溫度：" + temp + " °C\n💧 濕度：" + humi + " %";
  
  var payload = {
    "to": TARGET_USER_ID,
    "messages": [{
      "type": "text",
      "text": message_text
    }]
  };
  
  var options = {
    "method": "post",
    "headers": {
      "Content-Type": "application/json",
      "Authorization": "Bearer " + LINE_ACCESS_TOKEN
    },
    "payload": JSON.stringify(payload),
    "muteHttpExceptions": true
  };
  
  // 4. 執行發送動作至 LINE 伺服器
  UrlFetchApp.fetch(line_api_url, options);
  
  return ContentService.createTextOutput("Success: Logged to Sheet and Sent to LINE.");
}

樹莓派用戶端：Python 傳送程式碼
import time
import requests
import board
import adafruit_dht

# 📢 貼上你的相關參數
GAS_WEBAPP_URL = "https://script.google.com/macros/s/XXXXX/exec"
SPREADSHEET_ID = "貼上你檢索到的Google試算表ID亂碼"

# 連接至實體第 7 針腳 (GPIO 4)
dhtDevice = adafruit_dht.DHT11(board.D4)

print("樹莓派 方案 A 系統啟動...")

try:
    while True:
        try:
            temperature = dhtDevice.temperature
            humidity = dhtDevice.humidity
            
            if humidity is not None and temperature is not None:
                payload = {
                    "spreadsheet_id": SPREADSHEET_ID,
                    "temperature": round(temperature, 1), 
                    "humidity": round(humidity, 1)
                }
                res = requests.post(GAS_WEBAPP_URL, json=payload, timeout=10)
                print(f"【方案 A】上傳成功！回應: {res.text}")
                
            time.sleep(600)  # 每 10 分鐘自動傳送一次
        except RuntimeError as error:
            time.sleep(2.0)
            continue
        except Exception as error:
            dhtDevice.exit()
            raise error
except KeyboardInterrupt:
    dhtDevice.exit()
