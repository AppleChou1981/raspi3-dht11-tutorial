樹莓派 3 物聯網專案實作教學

DHT11 溫溼度數據收集：同步上傳 Google 試算表 & LINE Bot 即時推播 (GAS 方案 A)

💡 架構導讀： 本教學採用最乾淨獨立的「方案 A」。由樹莓派作為本地主控核心，讀取 DHT11 後，同時分頭呼叫 Google API 進行歷史資料存檔，並呼叫 LINE API 進行即時警報通知。此架構具備低耦合度、方便在終端機直接 Debug，且完全不佔用 Google 雲端的 Fetch 每日配額。

階段一：Google 雲端後台設定 (GAS 網址生成)

1
建立雲端試算表

打開你的 Google 雲端硬碟，新增一張全新的「Google 試算表」。這張試算表將用來永久儲存樹莓派傳上來的歷史數據。

2
編寫 Apps Script 腳本

點擊試算表網頁上方選單的 「擴充功能」 > 「Apps Script」。將裡面的預設程式碼全部清空，並貼上以下負責資料接收的純淨版程式碼：

function doGet(e) {
  return handleRequest(e);
}

function doPost(e) {
  return handleRequest(e);
}

function handleRequest(e) {
  var temperature = e.parameter.temperature;
  var humidity = e.parameter.humidity;
  
  // 自動綁定目前腳本所在的試算表
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheet = ss.getSheets()[0];
  var now = Utilities.formatDate(new Date(), "Asia/Taipei", "yyyy-MM-dd HH:mm:ss");
  
  if (temperature !== undefined && humidity !== undefined) {
    // 依序寫入 A欄(時間), B欄(溫度), C欄(濕度)
    sheet.appendRow([now, temperature, humidity]);
    return ContentService.createTextOutput("Success: Data written to Sheet.");
  } else {
    return ContentService.createTextOutput("Error: Missing parameters.");
  }
}

貼上後，點擊上方的 「儲存 (磁碟片圖示 💾)」。

3
部署為網頁應用程式

這是讓外網設備（樹莓派）能夠順利傳入資料的關鍵設定：

  1. 點擊右上角的 「部署」 > 「新部署」。
  2. 點選左側「選取類型」旁的齒輪圖示，選擇 「網頁應用程式」。
  3. 專案主管：維持你的 Google 帳號。
  4. 誰可以存取：務必切換為「所有人 (Anyone)」。
  5. 點擊「部署」。首次部署會彈出授權視窗，請點擊「核對權限」 > 選擇你的帳號 > 點擊底部的「進階」 > 點擊「前往未命名專案(不安全)」 > 最後點擊「允許」。
👉 部署成功後，請完整複製畫面上的「網頁應用程式 URL」（網址結尾通常是 /exec），妥善備用。

階段二：LINE Developer 開發者憑證獲取

4
建立 LINE 官方帳號與啟用 API
  1. 前往 LINE Business 後台建立一個免費的官方帳號（例如命名為「樹莓派即時回報系統」）。
  2. 進入管理主頁後，點擊網頁右上方 「設定 (齒輪⚙️)」 > 左側選單「Messaging API」。
  3. 點擊 「啟用 Messaging API」，服務提供者 (Provider) 填入或選擇一個既有專案名稱。
5
撈取 LINE 關鍵金鑰字串

登入 LINE Developers Console 控制台，進入你對應的 Bot 設定區：

  • 獲取 User ID：在 Basic settings 頁籤最下方，複製 Your user ID（格式為 Uxxxxxxxx...）。
  • 獲取 Access Token：在 Messaging API 頁籤最下方，點擊 **Channel access token** 欄位旁的 「Issue」，複製整串複雜字串。
  • 加入好友：使用手機掃描該頁面的 QR Code，將這個官方帳號加為好友，否則手機會收不到推播。

階段三：樹莓派硬體控制與 Python 實作

6
硬體線路連接

請將你的 DHT11 溫溼度感測器與樹莓派 3 的 GPIO 進行對接：

  • 感測器 VCC ── 接樹莓派 Pin 1 (3.3V)
  • 感測器 GND ── 接樹莓派 Pin 6 (接地)
  • 感測器 Data (資料腳) ── 接樹莓派 GPIO 4 (實體板第 7 針腳)
7
修復環境核心 (解決 Unknown platform 錯誤)

由於新版作業系統（如 Bookworm 架構）停止支援舊型 Adafruit 庫，請在樹莓派終端機執行現代化套件更新：

sudo apt update
sudo apt install -y gpiod libgpiod-dev
pip3 install adafruit-circuitpython-dht --break-system-packages
8
部署 Python 雙向同步程式

在樹莓派終端機輸入 nano dht11_to_gsheet.py 建立腳本，並貼入下方完整整合代碼：

import time
import requests
import board
import adafruit_dht

# ==================== 1. 硬體與網路參數設定 ====================
# (1) 填入你在階段一複製的 Google Apps Script Web App 網址
GAS_URL = "請在此處貼上你的_Google_GAS_網址_exec"

# (2) 填入你在階段二複製的 LINE 憑證資訊
LINE_ACCESS_TOKEN = "請在此處貼上你的_LINE_Channel_Access_Token"
TARGET_USER_ID = "請在此處貼上你的_LINE_Your_user_ID"

# (3) DHT11 感測器接腳設定 (資料線接 GPIO 4 故寫 board.D4)
dhtDevice = adafruit_dht.DHT11(board.D4)

# ==================== 2. 功能函式定義 ====================

def send_to_google_sheet(temp, humi):
    """將溫溼度數據傳送到 Google 試算表存檔"""
    payload = {"temperature": temp, "humidity": humi}
    try:
        response = requests.get(GAS_URL, params=payload, timeout=10)
        if response.status_code == 200:
            print(f"【Google Sheet】同步成功！回應: {response.text}")
        else:
            print(f"【Google Sheet】發送失敗，狀態碼: {response.status_code}")
    except Exception as e:
        print(f"【Google Sheet】連線發生錯誤: {e}")

def send_to_line_bot(temp, humi):
    """將溫溼度數據直接發送到 LINE Bot 推播通知"""
    url = "https://api.line.me/v2/bot/message/push"
    current_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
    
    message_text = f"\n🌡️ 樹莓派溫溼度即時回報\n-------------------\n⏰ 時間：{current_time}\n°C 溫度：{temp} °C\n💧 濕度：{humi} %"
    
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {LINE_ACCESS_TOKEN}"
    }
    payload = {
        "to": TARGET_USER_ID,
        "messages": [{"type": "text", "text": message_text}]
    }
    try:
        response = requests.post(url, json=payload, headers=headers, timeout=10)
        if response.status_code == 200:
            print("【LINE Bot】訊息推送成功！")
        else:
            print(f"【LINE Bot】發送失敗，狀態碼: {response.status_code}, 回應: {response.text}")
    except Exception as e:
        print(f"【LINE Bot】連線發生錯誤: {e}")

# ==================== 3. 主程式執行迴圈 ====================
if __name__ == "__main__":
    print("樹莓派 DHT11 獨立雙向同步系統啟動...")
    try:
        while True:
            try:
                temperature = dhtDevice.temperature
                humidity = dhtDevice.humidity
                
                if humidity is not None and temperature is not None:
                    temp_fixed = round(temperature, 1)
                    humi_fixed = round(humidity, 1)
                    
                    print(f"\n[偵測到新數據] 溫度: {temp_fixed}°C | 濕度: {humi_fixed}%")
                    
                    send_to_google_sheet(temp_fixed, humi_fixed)
                    send_to_line_bot(temp_fixed, humi_fixed)
                
                # 每 10 分鐘 (600秒) 自動讀取發送一次
                time.sleep(600)

            except RuntimeError as error:
                # DHT11 對時序極度敏感，偶發讀取失敗為硬體物理特性，等待2秒重試即可
                print(f"讀取中發生微小錯誤 (重試中): {error.args[0]}")
                time.sleep(2.0)
                continue
            except Exception as error:
                dhtDevice.exit()
                raise error
    except KeyboardInterrupt:
        dhtDevice.exit()
        print("\n程式已由使用者手動終止。")

替換參數完畢後，按 Ctrl + O 儲存，再按 Ctrl + X 離開納諾編輯器。

9
專案啟動與驗證

在樹莓派終端機執行下方指令：

python3 dht11_to_gsheet.py

💡 成果驗證：此時終端機會定時刷新成功狀態。去檢查你的 Google 試算表是不是確實填入了資料，手機 LINE 也同步接收到了推播卡片，大功告成！
