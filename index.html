💡 架構導讀（方案 A）： 本教學採用「方案 A」。由樹莓派定時讀取 DHT11 後，轉化為 HTTP POST 請求發送給 Google Apps Script (GAS) 網頁應用程式。GAS 收到後扮演中控角色，負責寫入試算表，並同時呼叫 LINE Notify 或 Messaging API 傳送推播通知。此方案的優點是樹莓派程式碼極為簡潔。

2
雲端後台：Google Apps Script 指令碼

請在雲端硬碟建立 GAS 指令碼，並部署為「網頁應用程式（所有人皆可存取）」：

function doPost(e) {
  var params = JSON.parse(e.postData.contents);
  var temp = params.temperature;
  var humi = params.humidity;
  
  // 寫入試算表邏輯
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  sheet.appendRow([new Date(), temp, humi]);
  
  return ContentService.createTextOutput("Data logged successfully (Plan A)");
}
3
樹莓派用戶端：Python 傳送程式碼

在樹莓派中建立 dht11_gas_client.py，並填入發送邏輯：

import time
import requests
import board
import adafruit_dht

# 貼上你的 GAS 網頁應用程式部署網址 (Web App URL)
GAS_WEBAPP_URL = "https://script.google.com/macros/s/XXXXX/exec"

dhtDevice = adafruit_dht.DHT11(board.D4)

while True:
    try:
        temperature = dhtDevice.temperature
        humidity = dhtDevice.humidity
        if humidity is not None and temperature is not None:
            payload = {"temperature": temperature, "humidity": humidity}
            res = requests.post(GAS_WEBAPP_URL, json=payload, timeout=10)
            print(f"數據成功上傳 GAS! 回應: {res.text}")
        time.sleep(600)
    except RuntimeError:
        time.sleep(2.0)
        continue
